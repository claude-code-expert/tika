# rules/ - 자동 적용 규칙 디렉토리

특정 파일이나 디렉토리 작업 시 Claude Code가 자동으로 참조하는 규칙을 관리하는 디렉토리이다. 규칙은 작업 대상에 따라 자동으로 활성화되어, 일관된 코드 품질과 패턴을 유지한다.

## 하위 문서

### api-rules.md
`app/api/` 하위 파일 작업 시 적용되는 규칙. API Route Handler 작성 패턴, 요청 검증(Zod), 응답 형식, 에러 코드 체계, 금지 사항을 정의한다.

### component-rules.md
`src/components/` 하위 파일 작업 시 적용되는 규칙. 컴포넌트 분류(서버/클라이언트), 네이밍, Tailwind CSS 스타일링, 상태 관리 패턴, 접근성, 공통 UI 컴포넌트 사용법, 금지 사항을 정의한다.

## 새 규칙 추가 방법

1. 이 디렉토리에 `규칙이름.md` 파일을 생성한다.
2. 규칙이 적용되는 대상 파일/디렉토리 범위를 명시한다.
3. 작업 시 준수해야 할 패턴, 규칙, 금지 사항을 마크다운으로 기술한다.
