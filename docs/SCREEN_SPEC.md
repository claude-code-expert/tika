# SCR-001: 메인 보드 뷰

## 1. 화면 목적

칸반 보드를 통해 전체 티켓을 상태별로 분류하여 표시하고, 드래그앤드롭으로 상태를 전환한다. 사용자가 업무 현황을 한눈에 파악하고 관리하는 메인 화면.

## 2. 진입 경로

- URL: `/` (루트 페이지)
- 인증 후 자동 리다이렉트 (미인증 시 SCR-004 로그인 페이지로 이동)

4. **푸터** (`.card-foot`): 좌측=뱃지들, 우측=아바타

| 체크리스트 | 체크리스트 있음 | 진행률 텍스트 (예: 2/4) |
| 완료 | status === DONE | `.b-done` — 초록 배경, `.card.fin` (opacity 0.7) |

**카드 아바타**: 24px 원형, 배경색 + 이름 이니셜, 다수 시 `-4px` margin 겹침

### 드래그앤드롭 (FR-006)

| 단계 | 동작 | UI 피드백 |
|------|------|----------|
| 시작 | 카드 클릭 + 150ms 이상 유지 (또는 바로 드래그 시작) | cursor: grabbing, `shadow-card-drag`, rotate(3deg), opacity: 0.5 |

| API 실패 | 서버 에러 | 원래 위치로 롤백, 에러 토스트 표시 |

### 4-3. 상위 카테고리 캐스케이딩 (FR-010)

선택한 **티켓 타입에 따라** 상위 계층 드롭다운이 동적으로 결정된다.

**4단계 계층**: Goal > Story > Feature > Task

| 선택한 타입 | 표시되는 상위 드롭다운 | 설명 |
|------------|----------------------|------|
| Goal | (없음) | 최상위 — 상위 카테고리 선택 불필요 |
| Story | [Goal 선택 ▼] | Goal 하위에 배치 |
| Feature | [Goal 선택 ▼] → [Story 선택 ▼] | Goal 선택 → Story 활성화 |
| Task | [Goal 선택 ▼] → [Story 선택 ▼] → [Feature 선택 ▼] | Goal → Story → Feature 순 캐스케이딩 |

```
타입: Task 선택 시
[Goal 선택 ▼]  →  [Story 선택 ▼]  →  [Feature 선택 ▼]
     ↓                  ↓                   ↓
 Goal 변경 시       Story 목록           Feature 목록
 Story 리셋        Goal 하위만           Story 하위만
 Feature 리셋       Feature 리셋
```

- 타입 미선택 시 상위 카테고리 영역 전체 비활성화
- 타입 변경 시 상위 카테고리 초기화
- 상위 단계 변경 → 하위 단계 초기화
