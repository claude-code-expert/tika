## 1. 개인 워크스페이스 (Personal Workspace)

### 1-1. 인증 & 진입

- [x] Google OAuth로 로그인 후 `/onboarding`으로 리다이렉트된다
- [x] 온보딩 위저드에서 개인 워크스페이스가 자동 생성된다
- [x] 로그인 후 재방문 시 워크스페이스 보드로 바로 진입한다
- [x] 로그아웃 후 `/login` 페이지로 리다이렉트된다

### 1-2. 칸반 보드 — 기본

- [x] Backlog / TODO / IN_PROGRESS / DONE 4개 칼럼이 모두 표시된다
- [x] 각 칼럼 헤더에 현재 티켓 수 배지가 표시된다
- [x] 빈 칼럼에 "비어있음" 텍스트가 표시된다
- [x] 칼럼 배경색이 디자인 토큰대로 나온다 (TODO=파란, IN_PROGRESS=노란, DONE=초록)

### 1-3. WIP 한도 경고

- [x] IN_PROGRESS 칼럼 티켓이 3개 이하일 때 경고 배지가 없다
- [x] IN_PROGRESS 칼럼 티켓이 4개 이상이면 `⚠ 4/3` 형식의 주황색 배지가 표시된다
- [x] 배지에 마우스를 올리면 툴팁 "적정 칸반 티켓수 = 3, 현재 X건 초과"가 나타난다
- [x] 배지 title 속성 "WIP 한도 초과: 진행 중 업무가 3개를 초과했습니다"가 설정되어 있다
- [x] 드래그로 5번째 티켓을 IN_PROGRESS로 이동하면 즉시 `⚠ 5/3`으로 업데이트된다
- [x] 티켓을 IN_PROGRESS에서 다른 칼럼으로 이동해 3개 이하가 되면 경고 배지가 사라진다

### 1-4. 티켓 생성

- [x] "새 티켓" 버튼 클릭 시 생성 폼/모달이 열린다
- [x] 제목 없이 제출하면 "제목을 입력해주세요" 에러 메시지가 표시된다
- [x] 제목 101자 입력 후 제출하면 유효성 에러가 표시된다
- [x] 제목 1~100자 입력 시 정상 생성된다
- [x] 타입 선택: GOAL / STORY / FEATURE / TASK 중 선택 가능하다
- [x] 우선순위 선택: LOW / MEDIUM / HIGH / CRITICAL 중 선택 가능하다
- [x] 마감일 형식 `YYYY-MM-DD`이 아닌 값(예: `2026/02/24`) 입력 시 "날짜 형식이 올바르지 않습니다 (YYYY-MM-DD)" 에러가 표시된다
- [x] 유효한 마감일 입력 시 정상 생성된다
- [x] 생성된 티켓이 BACKLOG 칼럼에 즉시 나타난다 (Optimistic UI)
- [x] 생성 API 실패 시 티켓이 롤백된다

## 2. 팀 워크스페이스 — OWNER 기준

### 2-1. 팀 워크스페이스 생성

- [x] 온보딩 또는 WorkspaceSwitcher에서 "팀 워크스페이스 만들기"가 가능하다
- [x] 워크스페이스 이름 입력 후 생성된다
- [x] 생성 후 온보딩 위저드의 팀 워크스페이스 단계로 진입한다
- [x] 팀 워크스페이스가 이미 3개 있을 때 추가 생성 시도 시 `409 TEAM_WORKSPACE_LIMIT` 에러가 표시된다
- [x] 에러 메시지 "팀 워크스페이스는 최대 3개까지 생성할 수 있습니다"가 노출된다

### 2-2. 멤버 초대 (링크 방식)

- [x] `/workspace/[id]/members`에서 "초대 링크 생성" 버튼이 표시된다
- [x] 초대 링크 생성 후 클립보드 복사가 가능하다
- [x] 생성된 링크 `/invite/[token]` 접근 시 초대 수락 페이지가 표시된다
- [x] 수락 시 해당 사용자가 MEMBER 역할로 추가된다
- [x] 이미 멤버인 사용자가 초대 링크 접근 시 적절한 안내 메시지가 표시된다
- [x] 만료된 초대 링크 접근 시 에러 메시지가 표시된다

### 2-3. 참여 신청 검토

- [x] `/workspace/[id]/members`에서 PENDING 상태의 참여 신청 목록이 표시된다
- [x] "승인" 버튼 클릭 시 신청자가 MEMBER로 추가된다
- [x] "거절" 버튼 클릭 시 신청이 REJECTED 상태로 변경된다
- [x] 이미 APPROVED된 신청을 다시 승인 시도 시 `409 ALREADY_PROCESSED` 에러가 표시된다
- [x] 이미 REJECTED된 신청을 다시 거절 시도 시 `409 ALREADY_PROCESSED` 에러가 표시된다
- [x] 승인 후 신청자에게 JOIN_REQUEST_RESOLVED 알림이 발송된다
- [x] 거절 후 신청자에게 JOIN_REQUEST_RESOLVED 알림이 발송된다
- [x] OWNER에게 JOIN_REQUEST_RECEIVED 알림이 발송된다 (신청 시)

### 2-4. 역할 변경

- [x] 멤버 목록에서 특정 멤버의 역할(MEMBER ↔ VIEWER)을 변경할 수 있다
- [x] 역할 변경 후 해당 멤버에게 ROLE_CHANGED 알림이 발송된다
- [x] OWNER 자신의 역할은 변경할 수 없다 (마지막 OWNER 보호)
- [x] OWNER가 1명만 있을 때 OWNER를 다른 역할로 변경 시도 시 에러가 표시된다

### 2-5. 멤버 제거

- [x] 멤버 목록에서 특정 멤버를 제거할 수 있다
- [x] 제거 전 확인 다이얼로그가 표시된다
- [x] 제거 후 멤버 목록에서 해당 사용자가 사라진다
- [x] 제거된 멤버에게 MEMBER_REMOVED 알림이 발송된다
- [x] 워크스페이스에 OWNER가 1명뿐일 때 OWNER를 제거 시도 시 에러가 표시된다

### 2-6. 워크스페이스 설정 — OWNER 전용

- [x] 일반 설정에서 워크스페이스 이름 변경이 가능하다 (PATCH API 200)
- [x] 일반 설정에서 설명 변경이 가능하다
- [x] 워크스페이스 삭제: 삭제 전 이름 입력 확인이 필요하다
- [x] 정확한 이름 입력 후 삭제 시 워크스페이스가 삭제되고 이동한다
- [x] 잘못된 이름 입력 시 삭제가 진행되지 않는다
- [x] MEMBER/VIEWER가 일반 설정 변경 시도 시 `403 FORBIDDEN` 에러가 발생한다

### 2-7. 스프린트 관리 (OWNER 전용)

- [x] 스프린트 생성이 가능하다 (PLANNED 상태)
- [x] PLANNED 스프린트를 "활성화"하면 ACTIVE 상태가 된다
- [x] 이미 ACTIVE 스프린트가 있을 때 다른 스프린트 활성화 시도 시 `409` 에러가 표시된다
- [x] ACTIVE 스프린트를 "완료"하면 COMPLETED 상태가 된다
- [x] PLANNED 상태가 아닌 스프린트 활성화 시 `400` 에러가 표시된다
- [x] ACTIVE 상태가 아닌 스프린트 완료 시 `400` 에러가 표시된다
- [x] MEMBER가 스프린트 활성화/완료 시도 시 `403 FORBIDDEN` 에러가 발생한다
- [x] 스프린트 시작 시 모든 멤버에게 SPRINT_STARTED 알림이 발송된다
- [x] 스프린트 완료 시 모든 멤버에게 SPRINT_COMPLETED 알림이 발송된다

### 2-8. 팀 워크스페이스 — 티켓 한도

- [x] 팀 워크스페이스에서 티켓 1000개일 때 추가 생성 시도 시 `400 TICKET_LIMIT_EXCEEDED` 에러가 표시된다

### 3-3. 담당자 변경

- [x] 담당자 추가 시 해당 멤버에게 TICKET_ASSIGNED 알림이 발송된다
- [x] 담당자 제거 시 해당 멤버에게 TICKET_UNASSIGNED 알림이 발송된다

### 3-4. 댓글

- [x] 티켓에 댓글을 작성할 수 있다
- [x] 댓글 작성 시 티켓 담당자/관련자에게 TICKET_COMMENTED 알림이 발송된다

### 3-5. 참여 신청 (다른 사용자 기준)

- [x] 로그인 후 워크스페이스 검색에서 공개 팀 워크스페이스를 찾을 수 있다
- [x] "참여 신청" 버튼 클릭 시 신청이 접수된다 (201)
- [x] 이미 멤버인 워크스페이스에 신청 시 `409 ALREADY_MEMBER` 에러가 표시된다
- [x] 이미 PENDING 신청이 있는 경우 재신청 시 `409 ALREADY_REQUESTED` 에러가 표시된다
- [x] PERSONAL 타입 워크스페이스에 신청 시 `403 FORBIDDEN` 에러가 표시된다

### 3-6. 설정 — MEMBER 제한

- [x] 일반 설정 탭은 표시되지만 저장 시 `403 FORBIDDEN` 에러가 발생한다
- [x] 라벨 관리 탭은 접근 가능하다 (또는 읽기 전용 확인 필요)
- [x] 알림 설정 탭은 접근 가능하고 개인 설정 변경이 가능하다
- [x] 워크스페이스 삭제 버튼이 표시되지 않거나 비활성화되어 있다

## 4. 팀 워크스페이스 — VIEWER 기준

### 4-1. 접근 가능한 메뉴 (허용)

- [x] 사이드바에서 대시보드 메뉴가 표시되고 정상 조회된다
- [x] 사이드바에서 보드(Kanban) 메뉴가 표시되고 티켓 목록이 조회된다
- [x] 사이드바에서 WBS 메뉴가 표시되고 정상 조회된다
- [x] 사이드바에서 분석(Analytics) 메뉴가 표시되고 정상 조회된다
- [x] 알림 내역 페이지 `/notifications`에 접근 가능하다

### 4-2. 접근 불가 메뉴 (차단)

- [x] 사이드바에서 멤버(Members) 페이지 메뉴가 **표시되지 않거나** 접근 시 차단된다
  - (TeamSidebar `minRole: 'MEMBER'` 설정 + 서버 레벨 redirect 구현)

### 4-3. 티켓 생성 차단

- [x] 보드에서 "새 티켓" 버튼이 비활성화되거나 숨겨져 있다
- [x] 티켓 생성 API 직접 호출 시 `403 FORBIDDEN` 에러가 반환된다
- [x] 에러 메시지 "권한이 부족합니다"가 사용자에게 노출된다

### 4-4. 티켓 수정 차단

- [x] 티켓 상세 모달에서 제목/설명 수정 입력이 비활성화된다
- [x] 티켓 수정 API 직접 호출 시 `403 FORBIDDEN` 에러가 반환된다
- [x] 상태 변경(칼럼 이동) 드롭다운이 비활성화된다

### 4-5. 티켓 삭제 차단

- [x] 티켓 카드/상세에서 삭제 버튼이 표시되지 않거나 비활성화된다
- [x] 티켓 삭제 API 직접 호출 시 `403 FORBIDDEN` 에러가 반환된다

### 4-6. 드래그 앤 드롭 차단

- [x] VIEWER는 칼럼 간 드래그 앤 드롭이 작동하지 않는다
- [x] VIEWER는 칼럼 내 순서 변경 드래그가 작동하지 않는다

### 4-7. 설정 — VIEWER 제한

- [x] 일반 설정 저장 시 `403 FORBIDDEN` 에러가 발생한다
- [x] 알림 설정은 개인 설정이므로 변경 가능하다
- [x] 워크스페이스 삭제 버튼이 표시되지 않는다

### 4-8. 댓글 (VIEWER 차단)

- [x] VIEWER가 댓글 작성 시 `403 FORBIDDEN` 에러가 반환된다
  - (`POST /api/tickets/[id]/comments`에 `requireRole(MEMBER)` 가드 구현)
