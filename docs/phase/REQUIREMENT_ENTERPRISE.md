# Tika — Enterprise 티어 요구사항

> **버전**: Enterprise (v1.0.x)
> **레포지토리**: 별도 레포 (현재 tika 레포에서 개발하지 않음)
> **작성일**: 2026-03-11
> **전제**: Team Pro 티어의 모든 기능을 포함하며, 아래 기능을 추가로 제공한다.

---

## 1. 티어 개요

| 항목 | 내용 |
|------|------|
| 대상 | 중대형 기업, 보안 요구사항이 높은 조직 |
| 팀 워크스페이스 | **무제한** |
| 워크스페이스당 티켓 | **무제한** |
| 파일 첨부 | **무제한** (Team Pro: 10MB/건, 10개/티켓) |

| Team Pro 대비 추가 기능 | 아래 섹션 참조 |

## 2. 추가 기능 목록

### 2.1 셀프 호스팅 (온프레미스)

**개요**: 회사 자체 서버에 Tika를 설치하여 운영한다. 모든 데이터가 사내 인프라에 저장된다.

**초기 설치 플로우**:
1. Docker 이미지 또는 바이너리 배포
2. 설치 위저드 실행
3. DB 연결 정보 입력 (호스트, 포트, DB명, 계정, 비밀번호)
4. 관리자 계정 생성 (이메일, 이름, 비밀번호)
5. DB 마이그레이션 자동 실행
6. 설치 완료 → 로그인 페이지 이동

**지원 환경**:

| 항목 | 요구사항 |
|------|----------|
| 컨테이너 | Docker / Docker Compose |
| OS | Linux (Ubuntu 20.04+, CentOS 8+) |
| DB | PostgreSQL 14+ (외부 DB 연결) |
| 리버스 프록시 | Nginx / Traefik 지원 |
| HTTPS | 자체 인증서 또는 Let's Encrypt |

**처리 규칙**:
- 설치 위저드는 최초 1회만 실행 (이후 접근 차단)
- DB 연결 실패 시 명확한 에러 메시지 + 재입력 안내
- 환경변수 또는 `.env` 파일로 연결 정보 관리
- SaaS 버전과 동일한 기능 제공 (온프레미스 전용 제약 없음)

### 2.2 라이선스 관리

**개요**: 라이선스 키를 통해 정식 사용자를 인증하고 기능을 활성화한다.

**라이선스 발급 플로우**:
1. 관리자가 설치 후 라이선스 등록 페이지 접근
2. 회사 이메일 입력 → 인증 메일 발송
3. 메일에 포함된 라이선스 키 입력
4. 라이선스 검증 성공 → 전체 기능 활성화

**라이선스 데이터**:

| 필드 | 설명 |
|------|------|
| licenseKey | 고유 라이선스 키 (서명된 JWT 또는 암호화 문자열) |
| companyName | 회사명 |
| email | 등록 이메일 |
| maxUsers | 최대 사용자 수 |
| expiresAt | 만료일 |
| activatedAt | 활성화 시각 |

**처리 규칙**:
- 외부 인증 서버에서 온라인 검증 (최초 활성화 + 30일 주기 재검증)
- 오프라인 환경용 대체 검증 방식 제공 (시간 제한 오프라인 키)
- 라이선스 만료 30일 전부터 갱신 안내 배너 표시
- 만료 후: 읽기 전용 모드 (새 티켓 생성/수정 불가, 조회만 가능)
- 라이선스 키는 로컬 암호화 저장 (변조 방지)

**무료 체험 (Trial)**:

| 항목 | 제한 |
|------|------|
| 기간 | 설치일로부터 7일 |
| 워크스페이스 | 1개만 생성 가능 |
| 멤버 수 | 최대 5명 |
| 기능 | Team Pro 전체 기능 사용 가능 |

### 2.3 커스텀 워크플로우

**개요**: Team Pro의 자동화 워크플로우를 확장하여 조직 맞춤형 워크플로우를 정의한다.

**Team Pro 대비 확장 항목**:

| 항목 | Team Pro | Enterprise |
|------|:--------:|:----------:|
| 워크플로우 규칙 수 | 10개/워크스페이스 | 무제한 |
| 커스텀 상태 정의 | ❌ | ✅ |
| 승인 단계 워크플로우 | ❌ | ✅ |
| 외부 웹훅 액션 | ❌ | ✅ (REST API & 웹훅 연계) |
| 워크플로우 감사 로그 | ❌ | ✅ |

**커스텀 상태**:
- 기본 4단계(Backlog/TODO/IN_PROGRESS/DONE) 외 커스텀 상태 추가 가능
- 예: `CODE_REVIEW`, `QA`, `STAGING`, `RELEASED`
- 상태별 진입/이탈 조건 설정 가능

**승인 워크플로우**:
- 특정 상태 전환 시 지정된 멤버의 승인 필요
- 승인자 지정 (역할 기반 또는 개인 지정)
- 승인/거절 시 알림 발송

### 2.4 AI 도구 연동 (API Key 연동)

**개요**: 외부 AI 서비스 API Key를 등록하여 Tika 내에서 AI 기능을 활용한다.

**지원 AI 서비스**:

| 서비스 | 활용 예시 |
|--------|----------|
| Claude (Anthropic) | 티켓 설명 요약, 우선순위 제안, 관련 티켓 추천 |
| OpenAI | 티켓 설명 자동 생성, 태그 자동 추천 |

**기능 상세**:

| 기능 | 설명 |
|------|------|
| 티켓 설명 요약 | 긴 설명을 1-2문장으로 요약 |
| 우선순위 제안 | 제목/설명 기반 우선순위 자동 추천 |
| 관련 티켓 추천 | 유사한 티켓 목록 제안 |
| 티켓 자동 분류 | 타입(Goal/Story/Feature/Task) 자동 분류 제안 |
| 체크리스트 자동 생성 | 설명 기반 체크리스트 항목 제안 |

**처리 규칙**:
- API Key는 암호화하여 DB 저장 (평문 미보관)
- 조직 단위 API Key 등록 (워크스페이스 공유 또는 개인 등록 선택)
- 사용량 모니터링 (요청 수, 토큰 수)
- AI 기능 on/off 토글 (워크스페이스 설정)

### 2.5 REST API & 웹훅

**개요**: Tika 데이터를 외부 시스템과 연동할 수 있는 공개 REST API와 웹훅을 제공한다.

**REST API**:

| 항목 | 내용 |
|------|------|
| 인증 | API Key (Organization 레벨) |
| 형식 | JSON REST API |
| 버전 관리 | `/api/v1/` 접두사 |
| Rate Limit | 분당 300회 |
| 문서 | OpenAPI 3.0 (Swagger UI 제공) |

**주요 엔드포인트**:
- `GET /api/v1/workspaces` — 워크스페이스 목록
- `GET /api/v1/tickets` — 티켓 목록 (필터/페이지네이션)
- `POST /api/v1/tickets` — 티켓 생성
- `PATCH /api/v1/tickets/[id]` — 티켓 수정
- `GET /api/v1/members` — 멤버 목록
- `GET /api/v1/analytics` — 분석 데이터

**웹훅**:

| 이벤트 | 설명 |
|--------|------|
| `ticket.created` | 티켓 생성 시 |
| `ticket.updated` | 티켓 수정 시 |
| `ticket.status_changed` | 상태 변경 시 |
| `ticket.deleted` | 티켓 삭제 시 |
| `member.joined` | 멤버 참여 시 |
| `member.removed` | 멤버 제거 시 |

**처리 규칙**:
- 웹훅 URL 등록 + 시크릿 키로 서명 검증
- 실패 시 3회 재시도 (지수 백오프)
- 웹훅 발송 이력 및 상태 조회

### 2.6 고급 검색 & 퀵 필터

**개요**: 전체 워크스페이스에 걸친 강력한 검색 기능을 제공한다.

**고급 검색 기능**:

| 기능 | 설명 |
|------|------|
| 전체 텍스트 검색 | 제목 + 설명 + 댓글 전문 검색 |
| 크로스 워크스페이스 검색 | 여러 워크스페이스에 걸친 검색 |
| 필터 조합 | 우선순위 + 타입 + 담당자 + 라벨 + 마감일 + 상태 복합 필터 |
| 검색 결과 저장 | 자주 쓰는 필터 조합 저장 (퀵 필터) |
| 검색 히스토리 | 최근 검색어 자동 저장 |

**퀵 필터**:
- 사용자 정의 필터 조합 저장 (최대 20개)
- 워크스페이스별 또는 전체 범위 설정
- 사이드바에 퀵 필터 목록 표시

### 2.7 무제한 파일 첨부

**개요**: Team Pro의 파일 첨부 제한을 해제한다.

| 항목 | Team Pro | Enterprise |
|------|:--------:|:----------:|
| 파일당 최대 크기 | 10 MB | 무제한* |
| 티켓당 최대 파일 수 | 10개 | 무제한 |
| 총 스토리지 | 제한 있음 | 별도 계약 |

*셀프 호스팅 환경에서는 자체 스토리지 용량에 따름

### 2.8 워크스페이스 제한 없음

**개요**: 워크스페이스 및 티켓 수 제한이 없다.

| 항목 | Workspace | Team Pro | Enterprise |
|------|:---------:|:--------:|:----------:|
| 팀 워크스페이스 수 | 3개 | 100개 | **무제한** |
| 워크스페이스당 티켓 | 1,000개 | 무제한 | **무제한** |
| 파일 첨부 | ❌ | 10MB/10개 | **무제한** |
| 멤버 수/워크스페이스 | 제한 없음 | 제한 없음 | **무제한** |
