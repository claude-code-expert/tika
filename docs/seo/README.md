# 🚀 Tika SEO & GEO Optimization Project

이 프로젝트는 **Tika** 프로젝트의 검색 엔진 최적화(SEO), 생성형 엔진 최적화(GEO), 그리고 웹 접근성(A11y)을 극대화하기 위한 가이드라인과 구현 샘플을 관리합니다. 특히 공개 서버에 배포되는 개별 위키 및 문서 페이지의 검색 노출 빈도와 상위 랭킹 진입을 목표로 합니다.

## 🛠️ 주요 적용 가이드 (Key Pillars)

### 1. 가이드라인 적용 (`SEO_GUIDE.md`)

모든 프론트엔드 컴포넌트와 문서 배포 시 다음의 7가지 원칙을 반드시 준수합니다.

- **시맨틱 HTML**: `<div>` 남발을 지양하고 `<main>`, `<article>`, `<nav>` 등 의미에 맞는 태그 사용.
- **메타데이터 제어**: Next.js Metadata API를 통한 동적 `title`, `description` 설정 (키워드 전진 배치).
- **기술적 SEO**: `sitemap.xml`, `robots.txt` 자동화 및 하이픈(`-`) 기반의 클린 URL 유지.
- **성능 최적화**: Core Web Vitals(LCP 2.5s 미만, INP 200ms 미만) 지표 준수 및 AVIF/WebP 이미지 사용.
- **구조화 데이터**: 검색 엔진이 정보를 구조적으로 이해하도록 `JSON-LD` (TechArticle 등) 적극 활용.
- **접근성(A11y)**: 모든 인터랙티브 요소에 ARIA 속성 적용 및 키보드 접근성 보장.
- **GEO(AI 최적화)**: AI 엔진(Perplexity, SearchGPT 등)이 정보를 요약하기 좋게 **TL;DR** 섹션 구성.

### 2. 컴포넌트 구현 (`WikiPage.tsx`)

개별 문서(Markdown, HTML, Wiki)가 배포될 때, 각 페이지는 독립적인 검색 색인이 가능하도록 설계되었습니다.

- **Dynamic Metadata**: 각 문서의 제목과 요약을 자동으로 Meta 태그에 주입하여 검색 결과 클릭률(CTR) 개선.
- **Rich Snippets**: 구글 검색 결과에서 별점, 작성자, 요약 등이 노출되도록 스키마 주입.
- **Internal Linking**: 고립된 페이지(Orphan Page) 방지를 위한 관련 문서 유기적 연결.

## 📝 활용 방법

1.  **가이드 숙지**: 새로운 컴포넌트를 만들 때 `SEO_GUIDE.md`를 체크리스트로 활용합니다.
2.  **페이지 생성**: `WikiPage.tsx`의 구조를 참고하여 `Metadata` 섹션과 `JSON-LD` 스크립트가 포함되었는지 확인합니다.
3.  **검증**: 배포 후 Google Search Console 및 Lighthouse를 통해 SEO 점수와 색인 여부를 모니터링합니다.
