## Schema Changes

### 1. `users` table — Add `type` column

**Change type**: ALTER TABLE (add column)

```typescript
// In src/db/schema.ts — add to users pgTable definition:
userType: varchar('user_type', { length: 20 }),
// NULL       = onboarding not completed (first login)
// 'USER'     = personal mode selected
// 'WORKSPACE' = workspace mode selected
```

**Drizzle schema addition:**
```typescript
export const users = pgTable('users', {
  id: text('id').primaryKey(),
  email: varchar('email', { length: 255 }).notNull().unique(),
  name: varchar('name', { length: 100 }).notNull(),
  avatarUrl: text('avatar_url'),
  userType: varchar('user_type', { length: 20 }),  // ← NEW: NULL | 'USER' | 'WORKSPACE'
  createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
});
```

**Migration SQL** (generated by drizzle-kit):
```sql
ALTER TABLE "users" ADD COLUMN "user_type" varchar(20);
```

**Data migration** (one-time, run before deploying):
```sql
UPDATE users SET user_type = 'USER' WHERE user_type IS NULL;
```
This ensures existing users skip re-onboarding.

### 3. `workspace_join_requests` — New table

**Change type**: CREATE TABLE

```typescript
// In src/db/schema.ts — new table:
export const workspaceJoinRequests = pgTable(
  'workspace_join_requests',
  {
    id: serial('id').primaryKey(),
    workspaceId: integer('workspace_id')
      .notNull()
      .references(() => workspaces.id, { onDelete: 'cascade' }),
    userId: text('user_id')
      .notNull()
      .references(() => users.id, { onDelete: 'cascade' }),
    message: text('message'),                    // optional join message from requester
    status: varchar('status', { length: 20 }).notNull().default('PENDING'),
    // 'PENDING' | 'APPROVED' | 'REJECTED'
    reviewedBy: integer('reviewed_by')
      .references(() => members.id, { onDelete: 'set null' }),
    reviewedAt: timestamp('reviewed_at', { withTimezone: true }),
    createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
  },
  (table) => [
    unique('join_requests_workspace_user_unique').on(table.workspaceId, table.userId),
    index('idx_join_requests_workspace_status').on(table.workspaceId, table.status),
  ],
);
```

**Migration SQL**:
```sql
CREATE TABLE "workspace_join_requests" (
  "id" serial PRIMARY KEY,
  "workspace_id" integer NOT NULL REFERENCES "workspaces"("id") ON DELETE CASCADE,
  "user_id" text NOT NULL REFERENCES "users"("id") ON DELETE CASCADE,
  "message" text,
  "status" varchar(20) NOT NULL DEFAULT 'PENDING',
  "reviewed_by" integer REFERENCES "members"("id") ON DELETE SET NULL,
  "reviewed_at" timestamptz,
  "created_at" timestamptz NOT NULL DEFAULT now(),
  UNIQUE("workspace_id", "user_id")
);
CREATE INDEX "idx_join_requests_workspace_status" ON "workspace_join_requests"("workspace_id", "status");
```

## New TypeScript Types (src/types/index.ts additions)

```typescript
// User onboarding type
export const USER_TYPE = {
  USER: 'USER',
  WORKSPACE: 'WORKSPACE',
} as const;
export type UserType = (typeof USER_TYPE)[keyof typeof USER_TYPE] | null;

// Join request status
export const JOIN_REQUEST_STATUS = {
  PENDING: 'PENDING',
  APPROVED: 'APPROVED',
  REJECTED: 'REJECTED',
} as const;
export type JoinRequestStatus = (typeof JOIN_REQUEST_STATUS)[keyof typeof JOIN_REQUEST_STATUS];

// Join request record
export interface JoinRequest {
  id: number;
  workspaceId: number;
  userId: string;
  message: string | null;
  status: JoinRequestStatus;
  reviewedBy: number | null;
  reviewedAt: string | null;
  createdAt: string;
}

// Join request with requester info (for OWNER view)
export interface JoinRequestWithUser extends JoinRequest {
  userName: string;
  userEmail: string;
  userAvatarUrl: string | null;
}

// Workspace search result (public info only)
export interface WorkspaceSearchResult {
  id: number;
  name: string;
  description: string | null;
  memberCount: number;
}
```

## Entity Relationships

```
users (1) ──────────────── (N) workspace_join_requests
   │                              │
   │                              └── workspaceId → workspaces
   │                              └── userId → users
   │                              └── reviewedBy → members
   │
   └─── (1:N) workspaces [via ownerId]
                 │
                 └── isSearchable: bool  ← NEW
                 └── type: PERSONAL | TEAM
```

---

## State Machine: Join Request Status

```
[PENDING] ──APPROVE──→ [APPROVED]  (member INSERT happens in same transaction)
[PENDING] ──REJECT──→  [REJECTED]
[APPROVED] → (terminal)
[REJECTED] → (terminal)
```
